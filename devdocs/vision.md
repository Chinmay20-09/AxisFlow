# AxisFlow Core Vision Document
## Version 1.0

> This document exists to prevent feature drift, overengineering, and loss of the original philosophy behind AxisFlow.

---

# What AxisFlow Is

AxisFlow is a:

- minimalist finance tracking app
- focused on clarity over complexity
- designed to reduce financial stress
- enhanced with quiet AI-powered insights

It is NOT:
- a bloated finance superapp
- a social platform
- an AI chatbot app
- a banking replacement
- a crypto/investment ecosystem

---

# Core Philosophy

AxisFlow should feel:

- calm
- intelligent
- lightweight
- intentional
- visually clean
- emotionally non-overwhelming

The app should help users:
- understand spending
- notice habits
- gain awareness
- improve gradually

without making them anxious.

---

# Primary Product Goal

The main goal of AxisFlow is:

> Helping users become more financially aware through minimal friction and intelligent insights.

NOT:

# Design Principles

## 1. Minimal First
Every screen should ask:

> "Can this be simpler?"

Avoid:
- clutter
- too many buttons
- excessive colors
- feature overload

---

## 2. Calm UI
The interface should feel:
- smooth
- breathable
- distraction-free

Users should not feel mentally exhausted after using the app.

---

## 3. AI In Background
AI should:
- analyze patterns
- summarize behavior
- provide light guidance

AI should NOT:
- dominate the interface
- behave like a chatbot
- constantly interrupt users

The AI exists to support awareness, not replace thinking.

---

## 4. Insights Over Data
Raw data is secondary.

Meaningful insights are primary.

Good:
- "Food spending increased 24% this week."

Bad:
- giant unreadable dashboards.

---

## 5. Frictionless Tracking
Adding transactions should be:
- fast
- smooth
- intuitive

The app succeeds when users consistently use it.

---

# Non-Negotiable Product Rules

## Never Build Features Just Because They Sound Cool

Every feature must answer:
1. Does this reduce friction?
2. Does this improve awareness?
3. Does this fit minimalism?
4. Would users actually use this weekly?

If not:
Do not build it.
